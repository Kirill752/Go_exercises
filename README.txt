1)Чтобы скомпилировать программу, имеющую сложную стркутуру файлов без ошибки на подобие этой: 
"./exercises" is relative, but relative import paths are not supported in module mode"
Необходимо в командной строке сначала собрать проект при помощи команды:

GO111MODULE=auto go build -o program main.go

"GO111MODULE" — это переменная среды, которую можно установить при использовании go для изменения способа импорта пакетов Go. 
Одна из первых болевых точек заключается в том, что в зависимости от версии Go его семантика меняется.

Команда "go build" соберет программу в бинарный файл program
2)Чтобы скомпилировать программу, состающую их нескольких файлов .go необходимо их собирать вместе при помощи go build
Пример: если есть 2 файла в package main в одной директории (main.go и findSQRT.go) то их собираем следующим образом

go build -o program findSQRT.go main.go
